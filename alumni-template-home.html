Alumni Template: Home

A Guide for iModules Templating - June 2014

This template is similar to the Sidebar template except that it has some additional sections filled in. The differences include:

  • No Page Title Section
  • The image banner comes before priority1and2
  • Separate CSS file to modify the Image Banner
  • Different Fields

Page Sample

  • Filled in Content

Image Banner

The image banner is modified through CSS. It was recommended that the CMS admin would change the image through the Overwrite CSS file. To make it easier for the CMS admin and to reduce the risk of unwanted CMS changes, I split off the relevant code into a separate CSS file. Ideally the two CSS files would be combined and compressed server side. The URL's will need to be updated based on file location.

News Grid

A news grid of 3 items follows the image banner. What's the best way to integrate this into the iModules CMS? These items will exist elsewhere in Imodules, is there a checkbox to make them display on the homepage? If not, let's break this up into fields:

  • Item 1 link
  • Item 1 img
  • Item 1 Title
  • Item 2 link
  • Item 2 img
  • Item 2 Title
  • Item 3 link
  • Item 3 img
  • Item 3 Title

Ads Grid

An ads grid of 4+ follows the news grid. What's the best way to integrate this into the iModules CMS? These items will exist elsewhere in Imodules, is there a checkbox to make them display on the homepage? If not, let's break this up into fields:

  • Item 1 link
  • Item 1 img
  • Item 1 Title
  • etc.

There is no set limit of amount of ads.

Flexible Content - Main

After the Ads Grid, put in a blank spot for HTML. This will give the CMS admins some flexibility if something comes up that doesn't fit into the grid. In general, this section will be empty.

There is no set limit of amount of ads.

Event Feed

An events feed follows the ads grid (but appears in the sidebar). What's the best way to integrate this into the iModules CMS? These items will all be iModules events. Is there a checkbox to make them display on the homepage? Let's set it up so that the feed is editable (formatting and number of events) without having to remake the template.

Twitter Feed

The Twitter feed goes below the Event Feed. Let's set it up so that the feed is editable in case the API changes.

Flexible Content - Sidebar

After the Twitter, put in a blank spot for HTML. This will give the CMS admins some flexibility if something comes up that doesn't fit into the grids. In general, this section will be empty.

Patterns (same as Full Width Template)

We have a Pattern Library detailing snippets of code to be used inside the templates. The library is being used for some channels using Drupal and some channels using iModules. In Drupal, for the most part, we have been able to extract the HTML into widgets so that content editors can employ patterns from the WYSIWYG. Perhaps you can help us do the same with iModules.
