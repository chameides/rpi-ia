Fields

  • Field 1: Title - Everthing inside the h1 tag.
  • Field 2: Subtitle (Optional Field) - Everthing inside the h2 tag. Can you set it up so that if empty: no empty h2?
  • Field 3: Main Content - Everything inside the article class="maincontent".
  • Field 4: Aside (Optional Field)- Everything inside the aside class="priority3". Can you set it up so that if empty: no empty aside?

Consistent Coding

I am coordinating the CSS across multiple CMSs, so it would be great if you can leave it intact. If you need to add CSS, could you do it either before or after my CSS? If you do need to change my CSS, please let me know and I'll try to keep everything consistent. Similarly, if you need to change any of the HTML selectors or classes, let me know. I know that you will be adding code and that's great.

For now I'm delivering uncompressed CSS with comments as well as uncompressed HTML. The live site should be compressed.

Patterns

We have a Pattern Library detailing snippets of code to be used inside the templates. The library is being used for some channels using Drupal and some channels using iModules. In Drupal, for the most part, we have been able to extract the HTML into widgets so that content editors can employ patterns from the WYSIWYG. Perhaps you can help us do the same with iModules.

Header and Footer Scripts

I have several scripts in the header and footer. It would be better to extract this and be able to edit them through the CMS. All scripts are site-wide.
